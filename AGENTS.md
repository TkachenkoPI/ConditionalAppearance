# AGENTS.md

Инструкции для агентов (OpenAI Codex, GPT-агенты, любые инструменты, читающие AGENTS.md).
Содержимое этого файла можно целиком вставить в поле Instructions при создании Custom GPT.

## Что за проект

Цветик — обработка 1С:Предприятие с текучим интерфейсом для условного оформления форм. Оборачивает `УсловноеОформлениеКомпоновкиДанных`, чтобы вместо десятков строк с `Элементы.Добавить()`, `ЛевоеЗначение` и `ВидСравнения` писать одну читаемую цепочку.

Проект 1C:EDT, исходники выгружены в XML/BSL:

- `Tsvetik/src/ExternalDataProcessors/Цветик/ObjectModule.bsl` — весь программный интерфейс
- `Tsvetik/src/ExternalDataProcessors/Цветик/Forms/Демо/Module.bsl` — демо-форма, Тест1–8, МегаТест и ТестВидимости
- `README.md` — пользовательская документация
- `Tsvetik/bin/*.epf` — результат сборки EDT, не исходник

`Tsvetik/DT-INF/PROJECT.PMF` содержит строку `Base-Project`, привязывающую проект к локальной конфигурации разработчика. Это машинозависимое значение — не коммить его изменение.

Файлы BSL в кодировке UTF-8, отступы табами, имена русские. В демо-форме часть процедур объявлена английскими ключевыми словами (`Procedure`/`EndProcedure`) — платформа допускает оба варианта, не переписывай их без нужды.

## Как использовать Цветик

Получение объекта:

```bsl
// внутри конфигурации, куда обработка скопирована
Цветик = Обработки.Цветик.Создать().Установить(УсловноеОформление);

// внутри самой обработки Цветик
Цветик = РеквизитФормыВЗначение("Объект").Установить(УсловноеОформление);

// как внешняя обработка
Цветик = ВнешниеОбработки.Создать(ПутьКФайлу).Установить(УсловноеОформление);
```

`Установить` обязателен и идёт первым: без него любой метод бросит исключение.

Порядок цепочки: `Установить → Очистить → Элемент → [ГруппаX → отборы → КонецГруппы] → оформление`. Каждый метод возвращает объект. Оформление применяется к последнему `Элемент()`, отборы — к текущей открытой группе.

```bsl
Обработки.Цветик.Создать().Установить(УсловноеОформление)
	.Очистить()
	.Элемент(Элементы.Реквизит2)
		.ГруппаИ()
			.Равно(Элементы.Реквизит2, 0)
			.НеРавно(Элементы.Реквизит3, Истина)
		.КонецГруппы()
		.ЦветФона(WebЦвета.Золотой);
```

Группа не обязательна — без неё отборы попадают прямо в отбор элемента (неявная ГруппаИ).

## Программный интерфейс

Основные: `Установить(УсловноеОформление)`, `Очистить()`, `Элемент(Имена)`, `Оформление(Свойство, Значение)`, `СгенерироватьКод()`

Группы: `ГруппаИ()`, `ГруппаИли()`, `ГруппаНе()`, `КонецГруппы()`

Отборы, все с сигнатурой `(Поле, Значение)`: `Равно`, `НеРавно`, `Больше`, `БольшеИлиРавно`, `Меньше`, `МеньшеИлиРавно`, `ВСписке`, `НеВСписке`, `ВИерархии`, `НеВИерархии`, `ВСпискеПоИерархии`, `НеВСпискеПоИерархии`, `Содержит`, `НеСодержит`, `Подобно`, `НеПодобно`, `НачинаетсяС`, `НеНачинаетсяС`, `Заполнено(Поле)`, `НеЗаполнено(Поле)`

Оформление: `ЦветФона`, `ЦветТекста`, `Шрифт`, `ГоризонтальноеПоложение`, `ВыделятьОтрицательные`, `ОтметкаНезаполненного`, `Видимость`, `Отображать`, `Доступность`, `ТолькоПросмотр`, `Текст`

`Элемент()` принимает имя строкой, список имён через запятую, элемент формы (берётся `.Имя`) или массив любого из этого. Отборы принимают строку пути к данным либо элемент формы, из которого берётся `.ПутьКДанным`.

Этот список — полный. Не придумывай методов, которых в нём нет.

## Ограничения, проверенные на практике

- **`Видимость` и `Отображать` применяются только к элементам таблицы** — к колонке или к самой таблице. На обычном поле формы вне таблицы отбор отрабатывает, а оформление молча не применяется: ни исключения, ни эффекта. Обычное поле прячется только кодом: `Элементы.Реквизит1.Видимость = Не Реквизит3;`
- **Метода `Формат()` не существует** — имя занято глобальной функцией платформы, объявить такой метод нельзя. Формат задаётся через `.Оформление("Формат", "ЧЦ=10; ЧДЦ=2")`.
- **Имена `WebЦвета.*` нельзя угадывать.** Неверное имя даёт `Поле объекта не обнаружено` во время выполнения, а не при проверке кода. Проверенные в проекте: `ЛососьСветлый`, `Золотой`, `Аквамарин`. Любой другой цвет задавай явным `Новый Цвет(R, G, B)`.
- **`Заполнено` и `НеЗаполнено`** второго аргумента не требуют: вид сравнения правое значение не использует.
- **Иерархические сравнения** имеют смысл только для ссылочных полей с иерархией.
- **`КонецГруппы()` без открытой группы** бросает исключение. Закрытие внешней группы возвращает к элементу оформления.
- **Опечатка в имени свойства в `Оформление()` проходит молча.** Проверить программно нельзя: `Строка(Параметр)` возвращает имя на языке интерфейса, в англоязычной конфигурации это `BackColor`, а не `ЦветФона`.

## Генерация кода

`СгенерироватьКод()` возвращает текст цепочки по текущему условному оформлению — для переноса «натыканного» мышью оформления в код.

Литералами выводятся строки, числа, булево, даты, web-цвета и горизонтальное положение. Всё остальное — `_ПодставьтеЗначение_`. Нераспознанный цвет, включая любой `Новый Цвет(R, G, B)`, заменяется на `WebЦвета.ЛососьСветлый`, поэтому цвета в результате генерации надо перепроверять.

## При правке самого Цветика

- В сообщениях об ошибках всегда указывай код языка: `НСтр("ru = 'Текст'", "ru")`. Без второго аргумента в неруссском интерфейсе `НСтр` вернёт пустую строку, и `СтрШаблон("", Значение)` упадёт с «Слишком много фактических параметров», маскируя настоящую ошибку.
- Новые методы-обёртки называй так, чтобы имя не совпадало с глобальной функцией платформы, иначе получишь синтаксическую ошибку.
- Не добавляй проверок, сравнивающих имена параметров оформления со `Строка(Параметр)` — они дают ложные срабатывания из-за языка интерфейса.
- Таблица web-цветов в `ObjectModule.bsl` набрана вручную по справке платформы. Ошибка в русском имени ломает генерацию для этого цвета, ошибка в английском уводит цвет в значение по умолчанию.

## Проверка

Автотестов нет, запуск требует платформы 1С. Доступная статическая проверка — валидация проекта в 1C:EDT (BSL Language Server): она ловит синтаксические ошибки, неопределённые методы и конфликты с именами платформы. После правок BSL прогоняй её и добивайся нуля ошибок. Перед проверкой поведения на форме проект нужно пересобрать в .epf.
