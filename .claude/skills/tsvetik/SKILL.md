# Цветик — условное оформление формы 1С

Текучий интерфейс поверх `УсловноеОформлениеКомпоновкиДанных`. Вместо десятков строк с `Элементы.Добавить()`, `ЛевоеЗначение`, `ВидСравнения` — одна читаемая цепочка.

## Получение объекта

Внутри конфигурации (рекомендуемый способ — обработка скопирована к себе):

```bsl
Цветик = Обработки.Цветик.Создать().Установить(УсловноеОформление);
```

Внутри самой обработки Цветик (например, в её форме):

```bsl
Цветик = РеквизитФормыВЗначение("Объект").Установить(УсловноеОформление);
```

Как внешняя обработка:

```bsl
Цветик = ВнешниеОбработки.Создать(ПутьКФайлу).Установить(УсловноеОформление);
```

`Установить` обязателен и идёт первым: без него любой метод бросит исключение.

## Порядок цепочки

```
Установить → Очистить → Элемент → [ГруппаX → отборы → КонецГруппы] → методы оформления
```

Каждый метод возвращает сам объект, поэтому цепочка не прерывается. Оформление применяется к последнему `Элемент()`, отборы — к текущей открытой группе.

```bsl
Обработки.Цветик.Создать().Установить(УсловноеОформление)
	.Очистить()
	.Элемент(Элементы.Реквизит2)
		.ГруппаИ()
			.Равно(Элементы.Реквизит2, 0)
			.НеРавно(Элементы.Реквизит3, Истина)
		.КонецГруппы()
		.ЦветФона(WebЦвета.Золотой);
```

Группа не обязательна — без неё отборы попадают прямо в отбор элемента (неявная ГруппаИ):

```bsl
	.Элемент(Элементы.Реквизит1)
		.Равно(Элементы.Реквизит3, Истина)
		.ЦветФона(WebЦвета.ЛососьСветлый);
```

## API

**Основные:** `Установить(УсловноеОформление)`, `Очистить()`, `Элемент(Имена)`, `Оформление(Свойство, Значение)`, `СгенерироватьКод()`

**Группы:** `ГруппаИ()`, `ГруппаИли()`, `ГруппаНе()`, `КонецГруппы()`

**Отборы** — все принимают `(Поле, Значение)`:
`Равно`, `НеРавно`, `Больше`, `БольшеИлиРавно`, `Меньше`, `МеньшеИлиРавно`, `ВСписке`, `НеВСписке`, `ВИерархии`, `НеВИерархии`, `ВСпискеПоИерархии`, `НеВСпискеПоИерархии`, `Содержит`, `НеСодержит`, `Подобно`, `НеПодобно`, `НачинаетсяС`, `НеНачинаетсяС`, `Заполнено(Поле)`, `НеЗаполнено(Поле)`

**Оформление:** `ЦветФона`, `ЦветТекста`, `Шрифт`, `ГоризонтальноеПоложение`, `ВыделятьОтрицательные`, `ОтметкаНезаполненного`, `Видимость`, `Отображать`, `Доступность`, `ТолькоПросмотр`, `Текст`

### Что принимает `Элемент()`

Имя строкой, список имён через запятую, элемент формы (берётся `.Имя`) или массив любого из этого:

```bsl
.Элемент("Реквизит1")
.Элемент("Реквизит1, Реквизит2, Состав")
.Элемент(Элементы.Реквизит1)
.Элемент(МассивЭлементовФормы)
```

Отборы принимают строку пути к данным (`"Объект.Товары.Цена"`) или элемент формы — из него берётся `.ПутьКДанным`.

## Ограничения — проверены на практике

**`Видимость` и `Отображать` работают только с таблицами.** Колонка таблицы или сама таблица — да. Обычное поле формы вне таблицы — нет: отбор отрабатывает, оформление молча не применяется, исключения не будет. Обычное поле прячется только кодом:

```bsl
Элементы.Реквизит1.Видимость = Не Реквизит3;
```

**Метода `Формат()` нет.** Имя занято глобальной функцией платформы, объявить такой метод нельзя. Только через основной метод:

```bsl
.Оформление("Формат", "ЧЦ=10; ЧДЦ=2")
```

**Не угадывай имена `WebЦвета.*`.** Неверное имя даёт `Поле объекта не обнаружено` во время выполнения, а не при проверке. Проверенные в этом проекте: `ЛососьСветлый`, `Золотой`, `Аквамарин`. Любой другой цвет задавай явным RGB:

```bsl
.ЦветФона(Новый Цвет(245, 245, 245))
```

**`Заполнено` и `НеЗаполнено`** второго аргумента не требуют — вид сравнения правого значения не использует.

**Иерархические сравнения** (`ВИерархии`, `НеВИерархии`, `ВСпискеПоИерархии`, `НеВСпискеПоИерархии`) работают только по ссылочным полям с иерархией. На числе или строке применять бессмысленно.

**`КонецГруппы()` без открытой группы** бросает исключение. Закрытие внешней группы возвращает к элементу оформления.

**Опечатка в имени свойства в `Оформление()` проходит молча** — ни исключения, ни эффекта. Проверить программно нельзя: `Строка(Параметр)` возвращает имя на языке интерфейса.

## Генерация кода

`СгенерироватьКод()` возвращает текст цепочки по текущему условному оформлению — удобно, когда оформление «натыкали» мышью и надо перенести в код.

Литералами выводятся строки, числа, булево, даты, web-цвета и горизонтальное положение. Остальное — `_ПодставьтеЗначение_`, дописывается руками. Нераспознанный цвет (в том числе любой `Новый Цвет(R, G, B)`) заменяется на `WebЦвета.ЛососьСветлый` — проверяй цвета в результате.

## При правке самого Цветика

Модуль объекта — `Tsvetik/src/ExternalDataProcessors/Цветик/ObjectModule.bsl`, демо-форма с примерами — `Forms/Демо/Module.bsl`.

В сообщениях об ошибках всегда указывай код языка: `НСтр("ru = 'Текст'", "ru")`. Без второго аргумента в неруссском интерфейсе `НСтр` вернёт пустую строку, и `СтрШаблон("", Значение)` упадёт с «Слишком много фактических параметров», маскируя настоящую ошибку.

Новые методы-обёртки над `Оформление()` называй так, чтобы имя не совпадало с глобальной функцией платформы — иначе синтаксическая ошибка.
